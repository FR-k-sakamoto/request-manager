# Infrastructure レイヤー 実装ガイドライン

本ステップでは、Domain層で定義したPort（インターフェース）の実装と、Prisma等の外部システムとの連携を行います。

## 1. 責務と配置先

- **責務**: Prismaを用いたDB操作、外部API通信、および Prismaモデルから Domain/DTO へのデータ変換。
- **配置先**: `src/infrastructure/repositories/**` (DBリポジトリ), `src/infrastructure/**` (その他外部サービス)。
- **Prisma操作の限定**: Prismaの利用は必ずこのディレクトリ内に留めます。

## 2. 絶対ルール (MUST)

- **Prisma型の隠蔽**: Prismaの型（自動生成される型）をUsecase層の引数や戻り値に漏らさないでください。必ずこの層で変換を完結させます。
- **マイグレーションの運用**: 通常運用で `prisma db push` は使用しません。スキーマ変更がある場合は `pnpm prisma:migrate:dev --name <変更内容>` を実行する手順をユーザーに案内してください。

## 3. このステップの完了条件（AIの振る舞い）

実装が完了したら、**コミット/Pushを行わず、ここで停止してください**。
ユーザに対し、以下の情報を提供して確認を求めてください。

1. 変更した Infrastructure / Prisma スキーマ ファイルの一覧。
2. DBスキーマに変更があった場合、マイグレーション実行の案内。
3. **コミットメッセージ案**: `feature(infra): 〇〇リポジトリの実装とPrismaスキーマ更新` など、この層に限定されたコミットを提案してください。
