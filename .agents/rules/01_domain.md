# Domain レイヤー 実装ガイドライン

本ステップでは、アプリケーションのビジネスルールとデータ構造（エンティティ・値オブジェクト）、および外部I/Oのインターフェース（Port）の定義のみを行います。

## 1. 責務と配置先

- **責務**: ビジネスルールの表現、計算ロジック、ドメイン制約の定義。
- **配置先**: `src/domain/**`。
- **Port定義**: `src/use-cases/**/ports/**` に、Usecaseが外部と通信するためのインターフェース（Repository等）を定義します。

## 2. 絶対ルール (MUST)

- **純粋なTypeScriptであること**: `next/*`, `react`, `next-auth`, `@mui/*`, `prisma` などのフレームワークやライブラリへの依存は一切禁止です。
- **I/Oの排除**: データベース通信やファイル操作などのI/O処理は一切記述せず、Portを経由する形をとります。

## 3. このステップの完了条件（AIの振る舞い）

実装が完了したら、**いかなる場合でもコミット/Pushを行わず、ここで停止してください**。
ユーザに対し、以下の情報を提供して確認を求めてください。

1. 変更した Domain / Port ファイルの一覧。
2. 実装したビジネスルールの要約。
3. **コミットメッセージ案**: `feature(domain): 〇〇のエンティティと計算ロジックを追加` など、この層に限定された細かい粒度のコミットを提案してください。
