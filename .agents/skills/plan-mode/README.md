# Plan Mode

このスキルは、要件が曖昧な依頼を「実装計画 + 影響分析 + 見積もり」に整理します。
コードは書かず、計画に徹します。

## 使うタイミング

- 仕様が固まっていない / 要件整理が必要
- 実装の影響範囲や工数感を知りたい
- 並列作業に分解したい

## 必要な入力

- 目標（何を達成したいか）
- 制約（KISS/MUI/クリーンアーキテクチャ等）
- 対象システム/機能の範囲
- 期限や優先度（あれば）

不足している情報は前提として明記します。

## 出力

- 日本語の計画（3-8ステップ）
- 影響分析（編集/追加予定ファイル）
- 規模見積（S/M/L）
- 並列ワークストリーム案
- リスク / 依存関係

## 使い方例

- 「この機能追加の実装計画を出して」
- 「影響ファイルと見積を教えて」
- 「並列で進められる単位に分けて」

## 注意点

- ここではコードは書かない
- MUI/KISS/クリーンアーキテクチャの制約を守る
- リポジトリに`AGENTS.md`があれば最優先で読む
