# テスト方針策定

Testing Trophy モデルに基づき、各モジュールのテストレイヤーを判断する。

| レイヤー    | 対象                                  | ツール/方法        |
| ----------- | ------------------------------------- | ------------------ |
| Static      | 型安全・コード品質                    | TypeScript, ESLint |
| Unit        | 複雑なビジネスロジック                | Vitest             |
| Integration | 複数モジュールの結合動作              | Vitest             |
| E2E         | 画面のメインシナリオ                  | Playwright         |
| 手動テスト  | E2Eでカバーしない異常系・エッジケース | 手動テスト仕様書   |

## 判断フロー

```
新機能/バグ修正
  ├─ 複雑なロジックがある？
  │   ├─ Yes → Unit テスト
  │   └─ No  → Integration/E2Eに含まれていれば不要
  ├─ 画面に関わる変更？
  │   ├─ メインシナリオ（正常系フロー） → E2E テスト
  │   └─ E2E対象外（異常系・エッジケース） → 手動テスト仕様書
  └─ バグ修正？ → 再発防止の回帰テストを追加（Unit / E2E / 手動テスト）
```

## Unit テスト対象の絞り込み基準

| 優先度   | 対象                   | 基準                                                 |
| -------- | ---------------------- | ---------------------------------------------------- |
| **必須** | 複雑なUsecase/Service  | 依存関数の呼び出しが3回以上、または条件分岐が3つ以上 |
| **必須** | 金額計算・日付処理     | バグ時のビジネスインパクトが大きい領域               |
| **推奨** | バリデーションロジック | 入力パターンが多いもの                               |
| **任意** | 単純なCRUD wrapper     | E2E/Integrationでカバー                              |

## 外部API（サードパーティ）のモック方針

- 外部API連携（GeminiAPI等）は通常のテストでは**モック化する**（安定性・速度・コスト・再現性のため）
- Unit/Integration: `vi.mock()` でAPIクライアントを差し替え
- E2E: `page.route()` でAPIレスポンスを固定
- 外部APIとの接続検証が必要な場合は、Smoke Test として別枠で少数のみ実行（PR時のCIには含めない）

## フォルダ配置ルール

- **Unit/Integration**: ソースの隣にコロケーション（`src/**/*.spec.ts`）
- **E2E**: `tests/e2e/*.e2e.spec.ts` に集約
- **E2Eシナリオ仕様書**: `docs/tests/scenarios/*.scenario.md`（メインシナリオ、常に最新を管理）
- **手動テスト仕様書**: `docs/tests/v1.x/`（E2Eでカバーしないパターンを記載し、手動で実行。`docs/tests/templates/scenario_template.md` に従いシナリオID付きテーブル形式で管理）

## 出力フォーマット

判断結果を以下のテーブル形式で出力する。

| モジュール/機能       | レイヤー   | 理由                     | 配置先                             |
| --------------------- | ---------- | ------------------------ | ---------------------------------- |
| order.usecase.ts      | Unit       | 条件分岐5つ、依存関数4つ | src/usecases/order.usecase.spec.ts |
| 注文画面 正常フロー   | E2E        | メインシナリオ           | tests/e2e/order.e2e.spec.ts        |
| 注文画面 決済エラー時 | 手動テスト | E2E対象外の異常系        | docs/tests/v1.x/order.md           |
| user.repository.ts    | 不要       | 単純なCRUD wrapper       | -                                  |

## CI 連携

| テスト種別                  | 実行タイミング        | 必須/任意              |
| --------------------------- | --------------------- | ---------------------- |
| Static (TypeScript, ESLint) | PR作成時              | 必須                   |
| Unit / Integration          | PR作成時              | 必須（マージブロック） |
| E2E                         | Nightly or リリース前 | 推奨                   |
