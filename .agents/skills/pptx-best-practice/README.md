# PPTX Best Practice

このスキルは、`.pptx` ファイルの読み取り・編集・新規作成・品質確認を
一連のワークフローとして実行するための手順を提供します。

## 使うタイミング

- 既存スライドからテキストを抽出して要約や再利用をしたい
- テンプレートを使って資料を更新したい
- 0からプレゼン資料を作成したい
- スライドのレイアウト崩れや視認性を検証したい
- FUNREPEATブランド準拠（白背景・ロゴ・コピーライトフッター）で資料を作りたい

## 必要な入力

- 対象ファイルパス（入力 `.pptx`）
- 実施したい作業（抽出 / 編集 / 新規作成 / 分割結合）
- 期待する出力（完成 `.pptx`、抽出テキスト、画像など）
- デザイン制約（ブランドカラー、フォント、トーン）があれば明示

FUNREPEAT向けのロゴはスキル同梱の固定パスを使用:

- `assets/funrepeat-logo.png`

## 出力

- 更新済み `.pptx` もしくは抽出テキスト
- 実行したコマンドと作業内容の要約
- QAで見つけた問題点と修正内容

## 依存関係

- `markitdown[pptx]`（テキスト抽出）
- `Pillow`（サムネイル生成）
- `pptxgenjs`（新規作成）
- `soffice`（PDF変換）
- `pdftoppm`（画像化）

## 注意点

- 初回出力を完成扱いにせず、必ず画像化して視覚QAを実施する
- プレースホルダ文字列の残存チェックを必須化する
- 余白、重なり、コントラスト、整列の不整合を重点確認する
- 依存不足が判明した場合は種類を問わずインストールしてから再実行する（不足のまま代替検証で完了扱いにしない）
- FUNREPEAT向けでは白背景基準・控えめ角丸・フッター著作権表記（年は生成時点の西暦）を必須化する
